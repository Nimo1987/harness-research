## 写作规范

1. 金字塔原理：结论先行，自上而下展开
2. 章节开头第一段就是核心结论
3. 每个段落第一句话就是该段落的论点
4. 每个段落 3-5 句话，信息密度要高
5. 段落之间要有逻辑过渡

## 标题规范

1. 所有标题必须是行动标题（Action Title），即完整的结论性句子
2. 禁止使用"市场概况"、"行业分析"这类描述性标签

## 格式规范

1. 输出纯 HTML 片段
2. 章节标题用 `<h2>`，子标题用 `<h3>`
3. 所有表格使用 HTML `<table>` 标签，必须有 `<caption>` 和 `<div class="source-note">`
4. 禁止 Markdown 表格语法
5. 关键数据标注：`<span class="confidence high/medium/low">[高/中/低置信度]</span>`
6. 引用：`<sup>[n]</sup>`
7. 如果分析素材中有 data_visualizations，生成对应的 HTML 表格

只输出纯 HTML 片段，不要包含 ```html 标记，不要输出任何其他内容。
